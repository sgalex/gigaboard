**Статус**: 🚧 В разработке

## Обзор

Данный документ описывает полный user journey от первого открытия GigaBoard до построения первого data pipeline с визуализациями. Система спроектирована для **минимизации барьера входа** и **немедленного получения результата**.

### Ключевые принципы onboarding

✅ **AI-first подход**: AI Assistant активен с первой секунды  
✅ **Минимум кликов**: 3 клика до первой визуализации  
✅ **Немедленный feedback**: результат виден за < 1 минуты  
✅ **Гибкие пути**: можно начать с файлов, API, БД или вообще без данных  
✅ **Прогрессивное раскрытие**: от простого к сложному

## Работа с AI Assistant и контекстом

### Автоматический контекст через выделение

```
Выделение узлов на канвасе = автоматический контекст AI

Без выделения:
AI видит всю доску (все узлы)

С выделением:
AI видит только выбранные узлы

Пример:
1. Пользователь выделяет 2 DataNodes (sales_q1, sales_q2)
2. AI Panel: "📍 Контекст: 2 узла"
3. Пользователь: "Сравни эти кварталы"
4. AI анализирует ТОЛЬКО выбранные 2 узла

См. полное описание в [AI_ASSISTANT.md](AI_ASSISTANT.md).
```

## UI/UX принципы onboarding

### 1. Минимум кликов до результата

**Цель: 3 клика до первой визуализации**

```
Путь:
1. [Загрузить файл] → выбор файла
2. Файл загружен → DataNode создан
3. [Создать визуализацию автоматически] → WidgetNode создан

Время: < 1 минута
```

### 2. AI-first подход

- AI Assistant **всегда активен** и подсказывает
- AI **предлагает действия** через кнопки
- AI **объясняет** что видит на канвасе
- AI **не требует ввода текста** (кнопки быстрых действий)

### 3. Прогрессивное раскрытие

**Уровень 1 (новичок):**
- Большие кнопки на welcome screen
- AI делает большую часть работы
- Автоматические визуализации

**Уровень 2 (опытный):**
- Контекстное меню DataNode
- Ручная настройка трансформаций
- Множественный выбор узлов

**Уровень 3 (эксперт):**
- SQL/Python редакторы
- Отладка трансформаций
- Версионирование и lineage

### 4. Визуальный feedback

```
Действие → Немедленный feedback

Загрузка файла    → Progress bar → DataNode fade-in
Создание виджета  → "Генерирую..." → WidgetNode + edge animation
Трансформация     → Progress → Новый DataNode + connecting edge
```

### 5. Никогда не показывать пустоту

- Пустой канвас → Welcome screen с кнопками
- Нет данных → Предложение демо/публичных данных
- Нет визуализаций → AI предлагает создать
- Ошибка → Предложение альтернативы
