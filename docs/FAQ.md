# AI Assistant Panel — FAQ (Frequently Asked Questions)

## ❓ General Questions

### Q1: Как AI знает контекст доски?
**A:** Backend сервис `BoardContextProvider` загружает все виджеты, их данные, связи и недавние изменения с доски. Это информация передаётся в системный prompt при каждом запросе к GigaChat. Таким образом AI понимает, что на доске находится.

Пример:
```
User: "Покажи тренд"
System Prompt: "На доске есть граф Sales (значения: 1000, 1200, 1500), таблица Regions..."
AI: "Вижу, у вас тренд возрастающий. Предлагаю добавить линейный график..."
```

---

### Q2: Может ли AI создавать новые виджеты самостоятельно?
**A:** Нет, AI только **рекомендует** создание виджетов через кнопку `[+ Добавить график]`. Только пользователь может применить рекомендацию кликом. Это обеспечивает:
- 🎯 Контроль пользователя (не "захламляет" доску)
- ⚡ Быстрое применение (одна кнопка)
- 📊 Прозрачность (видно, что именно будет создано)

---

### Q3: Как обрабатывается история диалога?
**A:** 
- **В памяти** (Redis): текущая сессия хранится в Redis с TTL 4 часа
- **В БД** (опционально): можно сохранять в PostgreSQL для аналитики
- **Контекст**: каждый новый вопрос учитывает предыдущие сообщения в сессии
- **Multi-turn**: AI помнит контекст между вопросами ("Что это означает?" → понимает, о чём речь)

---

### Q4: Безопасно ли отправлять данные с доски в GigaChat?
**A:** 
- ✅ Данные отправляются только в виде контекста (названия, типы, структура, не сами значения с ПД)
- ✅ JWT авторизация — только авторизованные пользователи
- ✅ HTTPS/WSS шифрование
- ✅ Rate limiting защищает от DDoS
- ✅ Логирование для аудита

**Примечание**: Если у вас чувствительные данные, можно отключить отправку полного контекста (feature flag).

---

### Q5: Какая максимальная длина вопроса?
**A:** Нет жёсткого лимита, но рекомендуется:
- 5-500 символов — оптимально
- Более 2000 символов — может быть медленнее
- Очень длинные вопросы обрезаются на клиенте (user warning)

---

### Q6: Что если AI дал неправильный ответ?
**A:** 
- 👎 Пользователь может нажать кнопку "Не помогло" (feedback)
- 📝 Feedback логируется для улучшения prompts
- 🔄 Пользователь может переформулировать вопрос
- 🚫 Можно отклонить рекомендацию (не нажимать кнопку действия)

## 🏗️ Architecture Questions

### Q7: Почему используется Redis для кэша?
**A:** Redis даёт:
- ⚡ Fast read/write (< 1ms)
- 📊 Pub/Sub для real-time событий
- 🛡️ Rate limiting эффективнее чем БД
- 🎯 Session management (автоудаление по TTL)
- 🔧 Легко масштабируется

---

### Q8: Может ли использоваться другой LLM вместо GigaChat?
**A:** Да, архитектура расширяема:
1. Создать новый `AIAdapter` (например, `OpenAIAdapter`)
2. Реализовать интерфейс `process_message()`
3. Заменить провайдер в контроллере AI Assistant
4. Тестировать

Пример:
```python
# Текущее
self.gigachat = GigaChatClient()

# Можно сделать
self.ai_provider = AIProviderFactory.create('gigachat')
# или
self.ai_provider = AIProviderFactory.create('openai')
```

---

### Q9: Как работает масштабирование при 1000+ пользователях на доске?
**A:** 
- 📡 Redis pub/sub распределяет события между инстансами
- ⚙️ Socket.IO поддерживает комнаты (room per board_id)
- 🔄 Несколько FastAPI инстансов могут работать параллельно
- 💾 PostgreSQL хранит данные, кэш в Redis

**Архитектура**:
- Load Balancer
- FastAPI Instances
  - FastAPI Instance 1 (+ Socket.IO)
  - FastAPI Instance 2 (+ Socket.IO)
  - FastAPI Instance 3 (+ Socket.IO)
    - Redis (shared pub/sub)
    - PostgreSQL (shared data)

---

### Q10: Почему AI Assistant в правой панели, а не модальное окно?
**A:** 
- 👁️ Видно одновременно с доской (удобнее анализировать)
- ⚡ Быстрый доступ (не нужно открывать модаль)
- 📊 Контекст (видишь и диалог, и результат на доске)
- 📱 Лучше на мобильных (свайп vs всплывающее окно)

## 🔐 Security Questions

### Q11: Кто может видеть историю диалога?
**A:** 
- 🔒 **История в сессии**: только текущий пользователь (в памяти браузера + Redis)
- 🔒 **История в БД** (если сохраняется): только владелец/collaborators доски
- 🔒 **Access Control**: проверка прав перед каждым запросом

---

### Q12: Что если кто-то попытается использовать API напрямую?
**A:** 
- 🛡️ JWT токен обязателен
- 🛡️ Rate limiting (10 сообщений/минуту)
- 🛡️ Валидация board_id (проверка прав)
- 🛡️ CORS политика (только с того же домена)
- 📝 Все запросы логируются

---

### Q13: Какие данные логируются?
**A:** 
- ✅ User ID, Board ID, Timestamp (для аудита)
- ✅ Message length, Response time (для метрик)
- ❌ Содержимое сообщений не логируется (приватность)
- ❌ Данные с доски не сохраняются в логах

---

## 🚀 Performance Questions

### Q14: Почему ответ AI занимает 1-2 секунды?
**A:** Время распределяется так:
- 📡 Network latency: ~100ms
- 🔍 Load board context: ~200-300ms
- 🧠 GigaChat inference: ~500-800ms
- 📝 Parse + cache: ~100ms
- **Итого**: ~1-1.3 сек (P50), <2 сек (P95)

**Оптимизация**:
- Кэширование частых вопросов (Redis)
- Предзагрузка контекста доски
- Батчинг запросов (если много пользователей)

---

### Q15: Что если доска содержит 10,000+ виджетов?
**A:** 
- ⚠️ Контекст может быть слишком большой
- **Решение**: Отправлять только релевантные виджеты
  - Видимые на текущем зуме
  - Последние изменённые
  - Связанные с текущей областью

```python
# Вместо всех 10000
all_widgets = board.widgets  # 10000 items

# Отправляем только релевантные
relevant_widgets = [
    w for w in all_widgets
    if w.is_visible() or w.is_recent()
][:100]  # Максимум 100 в контексте
```

---

### Q16: Есть ли кэширование ответов?
**A:** Да, трёхуровневое:
1. **Session cache** (Redis): 4 часа — текущая сессия
2. **Response cache** (Redis): 24 часа — ответы на типовые вопросы
3. **Browser cache** (localStorage): до перезагрузки

**Эффект**: Повторный вопрос ответится < 100ms

## 🎨 UX/UI Questions

### Q17: Как узнать, что AI обрабатывает запрос?
**A:** 
- ⏳ **Loading indicator**: мигающие три точки "..."
- ⌛ **Timeout**: если нет ответа > 5 сек, показать "Ошибка, повторить"
- 🎯 **Disable input**: поле ввода заблокировано до ответа

---

### Q18: Можно ли прерывать запрос?
**A:** 
- ✅ На клиенте: `AbortController` стандарт
- ⚠️ На сервере: запрос продолжит обрабатываться (но ответ не отправится)
- 🔧 Опционально: полная отмена на GigaChat (требует API)

---

### Q19: Что произойдёт, если выключить AI Assistant?
**A:** 
- 📌 Панель скроется с интерфейса (feature flag)
- 📝 Диалоги остаются в истории (не удаляются)
- ⚡ API endpoints остаются доступны (для интеграций)
- 🔄 Можно включить обратно в любой момент

---

### Q20: Поддерживает ли mobile версия AI Assistant?
**A:** 
- ✅ Да, адаптивная вёрстка
- 📱 На мобильных панель может быть:
  - Скрыта (свайп для открытия)
  - На весь экран (fullscreen mode)
  - Split screen (половина canvas, половина AI)
- 👍 Оптимизировано для touch (большие кнопки)

## 📊 Analytics & Monitoring Questions

### Q21: Какие метрики отслеживаются?
**A:** 
- 📈 Messages per day (engagement)
- ⏱️ Response time (P50, P95, P99)
- ✅ Action success rate (% применённых рекомендаций)
- ❌ Error rate (% ошибок)
- 👍 User feedback (like/dislike ratio)
- 🎯 Top questions (анонимизированные)
- 🌍 Geographic distribution

---

### Q22: Как контролировать стоимость GigaChat API?
**A:** 
- 💰 Rate limiting: максимум 10 сообщений/минуту на пользователя
- 🎯 Кэширование: повторные вопросы не идут в API
- 📊 Мониторинг: алерты если потребление > лимита
- 🔧 Quotas: можно установить лимит на борд/пользователя

**Пример**:
```python
# 1000 пользователей × 10 msg/min = 10,000 API calls/min max
# Если средний cost $0.001 per call = $10/min максимум
```

## 🛠️ Troubleshooting

### Q23: AI отвечает слишком общо (не учитывает контекст)
**A:** 
1. Проверить, загружается ли контекст доски
2. Увеличить детальность system prompt
3. Добавить больше примеров в prompt
4. Проверить, что board_id правильный

---

### Q24: Response time > 3 сек (медленно)
**A:** 
1. Проверить сетевую задержку (скорость интернета)
2. Проверить нагрузку на GigaChat API
3. Уменьшить размер контекста доски
4. Включить кэширование ответов

---

### Q25: "Rate limit exceeded" ошибка
**A:** 
- Пользователь отправил > 10 сообщений в минуту
- Решение: подождать 60 сек или использовать другой аккаунт
- Админ может увеличить лимит в конфиге
