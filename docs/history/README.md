# Архив документации (`docs/history/`)

## Executive Summary

Каталог **history/** хранит **снимки решений, отчёты о внедрении и устаревшие версии** спецификаций. Эти файлы **не являются** источником истины для текущего кода: они полезны для контекста «как пришли к текущему состоянию» и для поиска по обсуждениям.

**Актуальная архитектура и API**: [`../README.md`](../README.md) → в первую очередь [`../ARCHITECTURE.md`](../ARCHITECTURE.md), [`../MULTI_AGENT.md`](../MULTI_AGENT.md), [`../API.md`](../API.md).

---

## Как читать архив

| Признак в файле | Значение |
|-----------------|----------|
| Дата в заголовке / имени файла | Момент съёмки; поведение продукта могло измениться |
| «V1», «Legacy», «УСТАРЕВШЕЕ» | Заменено в V2 (см. ссылки в тексте или `MULTI_AGENT.md`) |
| Упоминания **CriticAgent**, **Developer Agent** (как отдельное ядро), **DataNode** вместо Source/Content | Историческая терминология; в коде: **QualityGateAgent** (ключ плана `validator`), **TransformCodexAgent** / **WidgetCodexAgent**, узлы **SourceNode/ContentNode** |
