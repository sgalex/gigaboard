## 📋 Что реализовано

### 1. CreateDataNodeDialog - Диалог создания источника данных

**Файл**: `apps/web/src/components/dialogs/CreateDataNodeDialog.tsx`

**Основные возможности:**

#### 🎯 Приоритетные типы источников данных:
1. **Файловая система** (по умолчанию) - AI автоматически распознает структуру
2. **API вызов** - полная конфигурация HTTP запросов
3. **CSV файл** - табличные данные
4. **JSON файл** - структурированные данные
5. **SQL запрос** (скоро) - выборка из БД
6. **Web scraping** (скоро) - извлечение с веб-страниц
7. **Streaming** (скоро) - потоковые данные

#### 📁 Работа с файлами:

**File Upload кнопка:**
- Выбор файла через стандартный диалог
- Автоматическое чтение содержимого
- Предпросмотр первых 1000 символов
- Отображение метаданных (имя, размер)

**Поддерживаемые форматы:**
- Текстовые: `.txt`, `.log`, `.md`
- Данные: `.csv`, `.json`
- Конфиги: `.xml`, `.yaml`, `.yml`

**Хранение данных:**
```typescript
{
  filename: "data.csv",
  file_type: "text/csv",
  file_size: 15360,
  raw_content: "...",
  needs_ai_processing: true  // Флаг для AI обработки
}
```

#### 🌐 API конфигурация:
- Endpoint URL
- HTTP Method (GET/POST/PUT/DELETE)
- Headers (JSON формат)
- Request Body (для POST/PUT)

### 2. Drag & Drop на канвас

**Файл**: `apps/web/src/components/board/BoardCanvas.tsx`

**Функциональность:**

#### Визуальные индикаторы:
- Overlay с синей рамкой при перетаскивании файла
- Иконка Database и подсказка о поддерживаемых форматах
- Backdrop blur эффект

#### Автоматическое создание DataNode:
```typescript
// При drop файла:
1. Чтение содержимого через FileReader
2. Определение типа по расширению (csv/json/other)
3. Создание DataNode с метаданными
4. Размещение на канвасе (случайная позиция)
```

#### Множественный drop:
- Можно перетащить несколько файлов сразу
- Каждый файл создает отдельный DataNode

## 🔍 Примеры использования

### Способ 1: Через кнопку "Добавить узел"
```
1. Нажать "Добавить узел" на канвасе
2. Выбрать "DataNode"
3. В диалоге выбрать тип "Любой файл"
4. Нажать "Выберите файл"
5. Указать файл data.csv
6. Увидеть предпросмотр
7. Нажать "Создать источник"
```

### Способ 2: Drag & Drop (быстрый)
```
1. Открыть папку с файлами в проводнике
2. Перетащить data.csv на канвас GigaBoard
3. Увидеть overlay с подсказкой
4. Отпустить мышь
5. DataNode создан автоматически!
```

### Способ 3: API источник
```
1. Нажать "Добавить узел" → "DataNode"
2. Выбрать "API вызов"
3. Endpoint: https://api.example.com/users
4. Method: GET
5. Headers: {"Authorization": "Bearer TOKEN"}
6. Нажать "Создать источник"
```

## 💡 Технические детали

### Архитектурные решения:

**1. AI processing flag:**
```typescript
data: {
  raw_content: "...",
  needs_ai_processing: true  // Сигнал для AI Agent
}
```
Когда AI Integration будет реализован, backend проверит этот флаг и запустит обработку через Reporter/Transformation Agent.

**2. File metadata хранение:**
Вместо немедленного парсинга, храним raw_content. Это позволит:
- AI выбрать оптимальный способ парсинга
- Сохранить оригинальные данные
- Переобработать при необходимости

**3. Conditional rendering:**
Каждый тип источника рендерит свои специфичные поля через `renderSourceSpecificFields()`. Легко добавлять новые типы.
