# Landing Page Simplification — 30 января 2025

## Executive Summary

Переработан контент landing page с фокусом на **ценность для пользователя** вместо технических подробностей. Убраны жаргонные термины (SourceNode, ContentNode, EXTRACT, data pipeline), заменены понятным языком и акцентом на результат.

## Цель изменений

**Проблема**: Landing page был слишком техническим — много упоминаний Source-Content Node Architecture, названий агентов на английском (Planner, Researcher), технических терминов (WebSocket, SSE, Kafka, data lineage, EXTRACT edge).

**Решение**: Упростить язык, сместить акцент с "как устроено" на "что вы получите".

## Принципы упрощения

1. **Убираем жаргон**: SourceNode, ContentNode, EXTRACT, data pipeline, data lineage
2. **Русифицируем**: Planner → Планировщик, Researcher → Сборщик данных
3. **Фокус на результат**: не "как устроено", а "что вы получите"
4. **Эмоциональные триггеры**: "Экономия времени", "Без программирования", "Простота использования"
5. **Конкретные примеры**: "Подключает все источники", "Строит графики и таблицу"
6. **Убираем числа**: "Latency < 150ms" → "Быстрый ответ", "10K записей/мин" → "мгновенно"
