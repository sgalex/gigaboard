# Доступ к проекту и доске (роли и авторизация)

## Executive Summary

В GigaBoard **владелец проекта** и **приглашённые участники** работают с одними и теми же досками и дашбордами, но с разными правами. У соавторов в таблице `project_collaborators` хранится роль: **`viewer`** (только просмотр), **`editor`** (изменение контента без управления участниками), **`admin`** (полный доступ к приглашениям и ролям соавторов, без отзыва прав у владельца). На уровне API разделены операции **просмотра** (`require_project_view_access`, `BoardService.get_board`) и **изменения** (`require_project_edit_access`, `BoardService.get_board_for_edit`). Детальные пути REST — в [`API.md`](./API.md) §3 и §«Поиск пользователей»; миграция схемы БД — `apps/backend/migrations/versions/17_project_collaborator_roles.py`.

## Роли

| Роль | Кто получает | Доски и дашборды | Пресеты, измерения, библиотека | Участники проекта |
|------|----------------|------------------|--------------------------------|-------------------|
| **Владелец** | `projects.user_id` | Полный контроль | Полный контроль | Приглашения, смена ролей соавторов, удаление проекта |
| **admin** (соавтор) | запись в `project_collaborators` | Как у editor | Как у editor | Приглашения и смена ролей соавторов; **нельзя** затронуть владельца |
| **editor** (соавтор) | по умолчанию для старых строк | Создание/изменение узлов, рёбер, трансформаций и т.д. | Создание/изменение | Только просмотр списка в диалоге «Совместный доступ» |
| **viewer** (соавтор) | новые приглашения по умолчанию «Просмотр» | Открытие, чтение данных, фильтры; **без** мутаций узлов/дашборда/пресетов | Только чтение | Только просмотр списка |

Сообщения об отказе: для операций записи при роли viewer — **403** с текстом о недостаточных правах; для операций шаринга при роли не owner и не admin — отдельное сообщение о праве управлять участниками.

## Backend: слой доступа

- **`ProjectAccessService`** (`apps/backend/app/services/project_access_service.py`):
  - `require_project_view_access` — любой доступ к проекту (владелец или запись в `project_collaborators`);
  - `require_project_edit_access` — владелец или роль не `viewer`;
  - `require_project_share_access` — владелец или `admin`;
  - `require_project_owner` — только владелец (удаление/переименование проекта и т.п.).
- **`BoardService.get_board`** — проверка **просмотра** проекта доски.
- **`BoardService.get_board_for_edit`** — просмотр + **`require_project_edit_access`** (мутации на доске).

Мутации **SourceNode**, **ContentNode** (включая трансформации и визуализации), **edges**, **widgets**, **comments** и связанные маршруты проверяют доску через **`get_board_for_edit`** там, где изменяется состояние. Чтение списков и узлов — через **`get_board`** или эквивалентные проверки по `content_id` / `board_id`.

**AI Assistant (доска)** (`POST /api/v1/boards/{board_id}/ai/chat`, удаление сессии чата): требуется **право на изменение** проекта доски (сообщения пишутся в БД). **GET** истории чата — достаточно **просмотра**.

**Поиск пользователей для приглашения**: `GET /api/v1/users/search?project_id=...&q=...` — только при **`require_project_share_access`** на указанном проекте (обязательный query `project_id`).

## Список проектов и карточка (UI)

Ответ **`GET /api/v1/projects`** (`ProjectWithBoardsResponse`) дополнительно содержит:

- **`my_access`** — `owner` | `viewer` | `editor` | `admin` для текущего пользователя;
- **`is_owner`** — `true`, если текущий пользователь — владелец проекта (`projects.user_id`);
- **`boards_count`**, **`dashboards_count`**, **`sources_count`**, **`content_nodes_count`**, **`widgets_count`**, **`tables_count`**, **`dimensions_count`**, **`filters_count`** — агрегаты для карточки на экране приветствия;
- **`access_user_count`** — число участников с доступом (владелец + соавторы);
- **`access_members_preview`** — до 10 записей `{ user_id, username, role }` для всплывающей подсказки на карточке.

В UI «Совместный доступ» при добавлении участника в тело **`POST /api/v1/projects/{id}/collaborators`** передаётся роль **`viewer`**; смена роли — через **`PATCH .../collaborators/{user_id}`** (владелец или admin).

На экране приветствия: счётчик участников и popover со списком; меню «Поделиться» — у **владельца** и **admin**; редактирование проекта — только у **владельца**; **удаление** — только у владельца, в выпадающем меню карточки пункт «Удалить» вынесен **в конец списка** и отделён **разделителем** от остальных действий.

**Экспорт и импорт (ZIP):** скачать архив проекта и создать новый проект из архива может пользователь с **правом на изменение** проекта (`require_project_edit_access`, не **viewer**). В UI подписи действий: **«Экспорт»** и **«Импорт проекта»** (импорт с экрана приветствия: основная кнопка в шапке и кнопка в пустом состоянии списка; отдельная дублирующая кнопка у заголовка «Мои проекты» не используется). См. [API.md](./API.md) §3 (`/export`, `/import-zip`).
