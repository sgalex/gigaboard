# Security Policy

## Поддерживаемые версии

Проект находится на стадии активной разработки. Текущая версия — **0.1.0-alpha**.

| Версия | Поддержка          |
| ------ | ------------------ |
| 0.1.x  | :white_check_mark: |

## Сообщения об уязвимостях

Если вы обнаружили уязвимость в системе безопасности, **не создавайте публичный issue**.

Вместо этого:

1. Отправьте описание уязвимости на email: **security@gigaboard.dev** (замените на реальный email)
2. Включите детальное описание проблемы
3. Если возможно, приложите шаги для воспроизведения
4. Предложите решение (если есть)

Мы постараемся ответить в течение **48 часов** и выпустить патч в течение **7 дней** для критических уязвимостей.

## Меры безопасности в проекте

### Backend
- Валидация всех входных данных через Pydantic schemas
- SQL injection защита через SQLAlchemy ORM
- JWT токены для аутентификации
- Sandbox execution для пользовательского кода
- Валидация HTML/JS кода виджетов (только CDN библиотеки, запрет eval/Function)

### Frontend
- XSS защита через React's default escaping
- CORS настройки для разрешённых origins
- Socket.IO аутентификация
- Secure cookies для сессий

### Infrastructure
- Environment variables для секретов (не коммитятся в git)
- Redis pub/sub для internal коммуникации агентов
- PostgreSQL с encrypted connections

## Известные ограничения

- **Alpha версия**: Проект в стадии разработки, возможны баги
- **Sandbox**: Python код выполняется с ограничениями, но 100% изоляция не гарантируется
- **AI Generated Code**: Код, генерируемый AI, проверяется, но может содержать неожиданное поведение
