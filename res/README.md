# `res/` — вспомогательные ресурсы

Каталог для **неисполняемых** артефактов, которые удобно хранить в репозитории рядом с кодом, но которые не являются частью приложения:

- шаблоны конфигурации для ВМ и инфраструктуры;
- демо-данные для сценариев, записи видео, онбординга.

## Содержимое

| Путь | Назначение |
|------|------------|
| `nginx-vm-gigaboard.conf` | Пример reverse-proxy nginx (порт 80 → Docker frontend на `127.0.0.1:3000`). |
| `demo-data/sales.csv` | Демонстрационный CSV для сценариев (см. `docs/VIDEO_DEMO_SCENARIO.md`). |

Исполняемые скрипты запуска и `docker-compose*.yml` остаются в **корне** репозитория.
